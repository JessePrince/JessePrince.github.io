Flow Models

Before flow matching, flow models are hard to train, it entails solving ODEs during training, which slows down the training process and may incur error when solving ODEs numerically. But still, it’s important to learn what’s before flow matching.

Diffeomorphisms and push-forward maps

Let’s denote the collection of functions f:Rm→Rnf:\mathbb{R}^m\rightarrow \mathbb{R}^nf:Rm→Rn with continuous partial derivatives of order r as Cr(Rm,Rn)C^r(\mathbb{R}^m, \mathbb{R}^n)Cr(Rm,Rn), the r’th order partial derivative is defined as

∂rfk∂xi1⋯∂xirk∈[1,2,…,n],ij∈[1,2,…,m]\frac{\partial^r f^k}{\partial x^{i_1}\cdots \partial x^{i_r}}\quad k\in [1,2,\dots,n], i_j\in [1,2,\dots,m] ∂xi1​⋯∂xir​∂rfk​k∈[1,2,…,n],ij​∈[1,2,…,m]

We simplify denote Cr(Rn):=Cr(Rn,R)C^r(\mathbb{R}^n):=C^r(\mathbb{R}^n,\mathbb{R})Cr(Rn):=Cr(Rn,R), e.g., C1(Rm)C^1(\mathbb{R}^m)C1(Rm) denotes continuously differentiable scalar functions (map from Rm\mathbb{R}^mRm to R\mathbb{R}R).

Definition. If functions ψ∈Cr(Rn,Rn)\psi\in C^r(\mathbb{R}^n,\mathbb{R}^n)ψ∈Cr(Rn,Rn) with ψ−1∈Cr(Rn,Rn)\psi^{-1}\in C^r(\mathbb{R}^n,\mathbb{R}^n)ψ−1∈Cr(Rn,Rn), we say the class of functions are CrC^rCr diffeomorphism.

So diffeomorphism poses two strong conditions: (1) the functions have r’th order derivative and map from Rn\mathbb{R}^nRn to Rn\mathbb{R}^nRn, (2) the function is invertible, and the invert functions also have r’th order derivative.

Now let’s consider Y=ψ(X)Y = \psi(X)Y=ψ(X), where X∼pXX\sim p_XX∼pX​ is a random vector and ψ:Rd→Rd\psi: \mathbb{R}^d\rightarrow \mathbb{R}^dψ:Rd→Rd is a C1C^1C1 diffeomorphism, we can calculate the PDF of YYY, which is pYp_YpY​, by Change of Variables formula

pY(y)=pX(ψ−1(y))∣det⁡∂yψ−1(y)∣p_Y(y) = p_X(\psi^{-1}(y))|\det \partial_y\psi^{-1}(y)| pY​(y)=pX​(ψ−1(y))∣det∂y​ψ−1(y)∣

where ∂yϕ(y)\partial_y\phi(y)∂y​ϕ(y) denotes the Jacobian matrix. We further denote the push-forward operator with the symbol ♯\sharp♯

[ψ♯pX](y):=pX(ψ−1(y))∣det⁡∂yψ−1(y)∣[\psi_{\sharp}p_X](y) := p_X(\psi^{-1}(y))|\det \partial_y\psi^{-1}(y)| [ψ♯​pX​](y):=pX​(ψ−1(y))∣det∂y​ψ−1(y)∣

The reason we call it push-forward is that in the context of flow models, ψ\psiψ describes the position after some period of time, so this function pushes forward (in time dimension) the distribution from XXX to YYY, YYY is the distribution after some period of time.

Generative Flow Models

Definition. A CrC^rCr flow is a time-dependent mapping ψ:[0,1]×Rd→Rd\psi:[0,1]\times \mathbb{R}^d\rightarrow \mathbb{R}^dψ:[0,1]×Rd→Rd implementing ψ:(t,x)→ψt(x)\psi:(t,x)\rightarrow \psi_t(x)ψ:(t,x)→ψt​(x).

The above flow is a Cr([0,1]×Rd,Rd)C^r([0,1]\times \mathbb{R}^d,\mathbb{R}^d)Cr([0,1]×Rd,Rd) function, such that the function ψt(x)\psi_t(x)ψt​(x) is a CrC^rCr diffeomorphism in xxx for all t∈[0,1]t\in[0,1]t∈[0,1].

Xt=ψt(X0)X_t = \psi_t(X_0)Xt​=ψt​(X0​)

We can prove that XtX_tXt​ is Markov, let 0≤t<s≤10\leq t<s\leq 10≤t<s≤1, then

Xs=ψs(X0)=ψs(ψt−1(ψt(X0)))=ψs∣t(Xt)X_s = \psi_s(X_0)=\psi_s(\psi_t^{-1}(\psi_t(X_0))) = \psi_{s|t}(X_t) Xs​=ψs​(X0​)=ψs​(ψt−1​(ψt​(X0​)))=ψs∣t​(Xt​)

where ψs∣t:=ψs∘ψt−1\psi_{s|t}:=\psi_s\circ \psi_t^{-1}ψs∣t​:=ψs​∘ψt−1​, so a state at sss, which is later than ttt, depends only on ttt.

Probability Path and the Continuity Equation

Definition. A time-dependent probability (pt)0≤t≤1(p_t)_{0\leq t\leq 1}(pt​)0≤t≤1​ is called a probability path.

We can say Xt∼ptX_t\sim p_tXt​∼pt​, and ptp_tpt​ can be obtained by the push-forward function

pt(x)=[ψt♯p](x)p_t(x) = [\psi_{t\sharp}p](x) pt​(x)=[ψt♯​p](x)

which basically pushes the distribution from an initial point to a distribution later at time ttt. We can formally define this as utu_tut​ generates ptp_tpt​ if Xt=ψt(X0)∼ptX_t=\psi_t(X_0)\sim p_tXt​=ψt​(X0​)∼pt​ for all t∈[0,1)t\in [0,1)t∈[0,1), the right end is opened to handle velocity not defined precisely at t=1t=1t=1.

If utu_tut​ really generates ptp_tpt​, they satisfy the following partial differential equation know as the Continuity Equation

ddtpt(x)+div(ptut)(x)=0(1)\frac{\mathrm{d}}{\mathrm{d}t}p_t(x)+\mathrm{div} (p_tu_t)(x)=0\tag{1} dtd​pt​(x)+div(pt​ut​)(x)=0(1)

we define the divergence here as

div(v)(x)=∑i=1d∂xivi(x)\mathrm{div} (v)(x) = \sum_{i=1}^d \partial_{x^i}v^i(x) div(v)(x)=i=1∑d​∂xi​vi(x)

for v(x)=(v1(x),…,vd(x))v(x)=(v^1(x),\dots, v^d(x))v(x)=(v1(x),…,vd(x)), this basically resembles the conventional divergence in field theory. With continuity equation, we have a theorem named mass conservation

Theorem. Let ptp_tpt​ be a probability path and u1u_1u1​ a locally Lipchitz integrable vector field. Then, the following two statements are equivalent:

  1. The continuity equation holds for t∈[0,1)t\in [0,1)t∈[0,1)
  2. utu_tut​ generates ptp_tpt​.

The continuity equation actually has a straightforward physical meaning, recall the Gaussian law or the Divergence Theorem

∫Ddiv(u)(x)dx=∫∂D⟨u(y),n(y)⟩dsy\int_D \mathrm{div}(u)(x)dx = \int_{\partial D}\langle u(y),n(y)\rangle ds_y ∫D​div(u)(x)dx=∫∂D​⟨u(y),n(y)⟩dsy​

which states that, the divergence inside a volume DDD equals the flux leaving DDD by orthogonally crossing its boundary ∂D\partial D∂D.

Now let’s rewrite Equation (1)

ddt∫Dpt(x)dx=−∫Ddiv(ptut)(x)dx=−∫∂D⟨pt(y)ut(y),n(y)⟩dsy\frac{\mathrm{d}}{\mathrm{d}t}\int_D p_t(x)dx = -\int_D\mathrm{div}(p_tu_t)(x)dx = -\int_{\partial D}\langle p_t(y)u_t(y),n(y) \rangle ds_y dtd​∫D​pt​(x)dx=−∫D​div(pt​ut​)(x)dx=−∫∂D​⟨pt​(y)ut​(y),n(y)⟩dsy​

So continuity equation is saying the rate of change of total probability mass in the volume DDD is the negative probability flux leaving the domain, where the probability flux is defined as jt(y)=pt(y)ut(y)j_t(y)=p_t(y)u_t(y)jt​(y)=pt​(y)ut​(y)

Training Flow Models

To train a flow model, we can still paramterize the velocity field as utθu_t^\thetautθ​, and the model should learn θ\thetaθ such that

p1θ≈qp_1^\theta \approx q p1θ​≈q

which means the distribution at t=1t=1t=1 is the target distribution.

We can ahchieve this simply using KL-divergence

L(θ)=DKL(q,p1θ)=−EY∼qlog⁡p1θ(Y)+C\mathcal{L}(\theta)=D_{KL}(q,p_1^\theta) = -\mathbb{E}_{Y\sim q}\log p_1^\theta (Y) + C L(θ)=DKL​(q,p1θ​)=−EY∼q​logp1θ​(Y)+C

where CCC is a constant.

To obtain log⁡p1θ(Y)\log p_1^\theta (Y)logp1θ​(Y), we can use (9) and (10) by setting ut=utθu_t = u_t^\thetaut​=utθ​ which is our neural network and x=Yx=Yx=Y, samples from target distribution.

As you can see, computing the loss requires simulation of Equation (9), this is inefficient, and solving ODE numerically poses errors which will make the gradients biased.
