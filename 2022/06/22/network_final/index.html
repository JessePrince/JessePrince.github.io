1 Topology*

拓扑描述了网络中节点是如何连接的

1.1 总线

在总线型结构中，所有的站点都连接到一个线性的传输媒介上。总线通信中信息向两边传输，所有站点都接收信息，当辨认到是传给自己的时候将其保存下来。媒介两端有吸收器(Terminator)吸收能量。

1.1.1 优点

  • 易于安装
  • 成本低
  • 对于小型网络来说是最好的选择

1.1.2 缺点

  • 连接的站点数量受到限制，连接的站点太多会导致效率下降
  • 依赖于总线媒介，如果总线媒介出错将导致整个网络出错
  • 安全性差，因为所有的站点都可以看到信息。

1.2 环形结构

站点由一组中继器(Repeater)组成，这些中继器点对点连接在一起，形成一个环。在这个环上，数据只在一个方向上传输(逆时针或顺时针).

1.2.1 优点

  • 不需要中心服务器来管理(对比星形)
  • 容易检错
  • 增减站点容易，因为只需要更改两个节点

2 Switching*

2.1 电路交换

电路交换指在两个通信站点之间建立一条专用的通信通路，这条通路是由网络节点之间的链路首位相接而成，在每条物理链路上都有该连接专用的逻辑信道。

电路交换通信包含以下三个步骤

  1. 电路建立：在通信之前，一方向另一方发送请求，请求建立一条专用的连接。
  2. 数据传送：此时两方可以进行通信，可以是数字的活着模拟的。
  3. 电路断连：经过通信之后，连接被终止，一般是两个站点中一方发起这个动作。断连的信号反向传播，取消分配给连接的专用资源。

2.1.1 Notes

为了在数据传输开始之前建立连接，通路中每一个结点必须保留一定容量，而且每个节点必须具备相应的内部交换能力来处理最开始的连接请求。另外，交换机要有分配资源，建立路由的能力。

优点

  1. 对于话音传输来说是不错的选择，因为话音传输对传输时延是不能容忍的。
  2. 除了经过传输链路时的传播时延，不存在其它时延。
  3. 透明性，一旦建立起一条电路，对于连接到网络上的两个站点而言，它们之间好像有一条直线连接，任何一端都不需要特殊的组网逻辑。

缺点

  1. 电路交换的效率可能非常低，对于话音来说，利用率可能比较高，因为一直有话音信号在传输，但是对于计算机通信来说，可能在大多数时间里连接都是空闲状态。
  2. 信息以恒定的数据率传输，因此接发设备必须以相同数据率接发，在连接不同的站点的时候，网络利用率就受到了限制。

2.2 分组交换

在分组交换中，数据被分割成一个个分组，并加上一些控制信息。网络处理分组流的方法有两种，数据报和虚电路。

  1. 数据报：每个分组被视为独立的，并在前进的道路上由每个节点决定下一个经过的节点，所以最后可能失序到达出口节点。因此，顺序可能被出口节点，也可能是终点的站点进行恢复。这种技术中，被独立对待的一个分组就称为一个数据报。
  2. 虚电路：在发送任何分组之前，都要现建立一个预定的路由，即双方之间的所有分组都沿相同的路由穿过网络。这种方式中节点不需要进行路由决策，但分组在每个节点仍然需要排队，仍然经过存储转发。

如果两个站点希望在很长一段时间内交换数据，使用虚电路是有好处的。对比数据报，它有以下优点

  1. 按序传输：它们在到达时仍然是以最初的顺序排列的。
  2. 更快：节点不需要进行路由决策，所以传输速度更快。

数据报的优点是：

  1. 数据报省掉了呼建立过程，因此如果一个站点只想传送一个或几个分组，那么使用数据报方式更加快捷。
  2. 灵活：如果某处发生了堵塞，节点可以灵活选择绕过堵塞的地方。虚电路因为决定了路由所以处理堵塞比较困难。
  3. 更可靠：一个节点出现故障，那么经过这个节点的虚电路都会丢失，而数据报方式可以寻找一条替代的路由。

2.3 面向连接与无连接

  • 面向连接的是可靠通信，通常有连接建立，信息传输，确认收到，超时重传，关闭连接的操作。有时候还需要让分组按序到达。
  • 无连接的是不可靠通信，通信一方只需要加入地址之后发送就可以了，不关心能不能送到。

2.4 程控交换机之后的软交换

3 TCP/IP 协议栈

TCP/IP将通信任务分割成五个相对独立的层(从下往上叙述)

  1. 物理层：数据传输设备(计算机)与传输媒体之间的物理接口，这一层主要定义传输媒体的特点，信号状态，数据率等特征。
  2. 网络接入层/数据链路层：为同一个网络相连的两个系统提供网络接入，并且为它们的数据选择路由以穿过网络。它是网络硬件的逻辑接口，可能面向流，面向分组，也可能提供可靠交付。[以太网，WIFI，ATM，帧中继]
  3. 网际层：如果两个设备连接到不同的网络，就需要一些进程负责将它们的数据跨越多个互联的网络，这就是网际层的功能，这一层使用IP来提供经过多个网络的路由功能。[IPv4,IPv6,ARP,ICMP,OSPF]
  4. 运输层(Host 2 Host层)：提供可靠的端到端服务，活着只是交付而不是可靠，提供这些功能最常用的就是TCP。这一层在端点之间传送数据，可能会提供差错控制，流量控制，可靠交付。[TCP,UDP]
  5. 应用层：用于支持各种不同用户应用程序的逻辑。

4 Fundamental of Protocol

网络层与层之间的通信(Peer 2 Peer)受限于一组规则或者规约的，特定格式的数据块完成，这些规则和规约就是协议。

4.1 协议的关键要素**

  • 语法 Syntax：考虑有关数据块的格式
  • 语义 Semantics：包含用语相互协调及差错处理的控制信息
  • 定时 Timing：包含速率匹配和数据排序

4.2 PDU

全称Protocol Data Unit，来自上一层的数据和控制信息结合在一起称为PDU。运输层为上一层的数据添加首部封装之后就称为运输PDU，运输PDU也叫报文段。

4.3 服务原语

邻接层之间的服务叫做原语(Primitive)，主要有以下几个

  • 请求 Request：服务用户发出原语指明要请求的服务
  • 指示 Indication：服务提供者发出原语，指示连接上的对等服务用户已调用了某程序，并提供了相关参数。向服务用户通知一个由提供者激活的动作。
  • 响应 Response：服务用户发出，用于应答或完成某些程序。
  • 证实 Confirm：服务提供者发出，用于应答或完成某些程序。

5 运输层中的TCP与UDP

5.1 TCP

全称Transmission Control Protocol，TCP提供的是可靠通信，是面向连接的。它有连接建立，数据传送，连接终止的机制。

TCP头部各字段

  • 源端口 Source Port：源TCP用户
  • 目的端口 Destination Port：目的TCP用户
  • 序号 Sequence Number：
  • 确认序号 Acknowledgement Number：包含了TCP实体希望接收的下一个数据八位组的序号。
  • 数据偏移量 Data Offset：首部中32位字的数量
  • 保留 Reversed：为将来使用保留
  • 标志 Flags：用来设定某些信号
  • 窗口 Window：流量控制的信用量分配
  • 校验和 Checksum：报文段中所有16bit字加上一个伪首部的二进制反码取模之和。
  • 紧急指针 Urgent Pointer：使接收方知道由多少紧急数据到来
  • 可变选项 Options：

5.2 UDP

全称User Data Protocol，为应用层提供无连接服务，是不可靠的，但这种协议开销小，可能很适合与某些情况。

连接建立和维护不适合与以下几个例子

  • 数据采集：实时数据采集的时候，一个偶然的数据单元丢失并不会造成严重的后果，因为很短时间内就会有下一个数据。
  • 向外分发数据：包括向用户广播消息，宣布一个新节点的加入或一个服务地址的改变。
  • 请求和响应：一个服务器向多个用户提供服务的时候，该服务使用应由应用层来协调，低层的连接是不必要的。
  • 实时应用：例如话音传输和遥测器，根本不需要面向连接的功能，例如重传。

UDP头部各字段

  • 源端口
  • 目的端口
  • 长度
  • 校验和

6 局域网/第二层

6.1 MAC*

MAC全称Medium Access Control，媒体接入控制。MAC协议用来保证有序且有效的使用网络传输带宽，它的功能如下

  • 在传输时，将数据和地址及差错检测字段组装成帧。
  • 在接收时，分解帧，并执行地址识别和差错检测。
  • 监管局域网传输媒体的接入。

而对应OSI中数据链路层的功能，还有一项功能被划分到LLC(Logic Link Control)层里面，即向高层提供接口并执行流量控制和差错控制。

MAC接收来自LLC层的数据块并执行相应功能，MAC的PDU就称为MAC帧。目前的MAC帧各有差异，但总归有以下几个字段

  • MAC控制：包含执行MAC协议功能所需的任意协议控制信息。
  • 目的MAC地址：帧在局域网中目的物理连接点
  • 源MAC地址：帧在局域网中的源物理连接点
  • LLC：来自上一层的LLC数据
  • CRC：循环冗余检验字段。

MAC会负责差错检测并丢弃出错的帧，LLC可以跟踪被正确接收的帧，并重传未正确接受的帧。

6.2 网桥与第二层交换机

6.2.1 网桥

为了连接其它局域网，以及实现广域网的互联，我们需要网桥或者路由器，一般来说网桥是里面简单的一种。

网桥被用在采用相同物理层和链路层协议的局域网之间，可以做到一种MAC格式到另一种MAC格式的映射。但为什么不全部划分到一个局域网里面呢？

  1. 可靠性：将一个组织内部的所有数据处理设备都连接到一个网络中，所存在的危险就是网络某个地方的故障可能导致所有的设备无法通信，通过使用网桥，可以将网络分割成各个单元分担风险。
  2. 性能：将设备划分到较小的局域网中，且使网内通信量远大于网间通信量。因为局域网性能随着网络设备数目增加而降低
  3. 地理位置：如果两个设备相距很远，需要两个独立的局域网。

网桥的功能

  1. 读取局域网A的所有帧，并接受站点地址在局域网B上的帧。
  2. 对局域网B使用MAC，在B上重传每个帧。
  3. 对从B到A的通信采用同样的动作。

Notes

  • 网桥不改变帧的内容格式，不额外封装
  • 网桥有足够的缓存空间以满足峰值需要
  • 网桥有寻址能力和路由选择
  • 一个网桥可以连接两个以上局域网

网桥工作在MAC层，两个局域网都使用同样的MAC层和LLC协议

需要路由的网桥
如果网桥连接了多个网桥，或者和其它通信网相连，那么网桥必须适当封装该帧，这个时候需要路由。
固定路由选择：使用预先定义好的静态路由表，但有些时候需要动态的维护

6.3 碰撞的解决方案

由于局域网中大家的通信都是争用的，所以说需要一种解决碰撞的方案，这样才能保证局域网的有效和高效，目前广泛采用的是IEEE802.3开发的基于以太网的局域网，使用了带冲突检测的CSMA/CD。

前身：ALOHA
站点可以在任何时间传输帧，然后进行一段时间的监听，时间长度就是两倍传输一个帧的时间再加上一个很小的固定时间增量。如果监听到对方发的确认，就视为成功，否则重传。

  • 利用率：18%

前身：时隙ALOHA
把信道上的时间组织成统一的时隙，时隙的长度等于帧的传输时间，所有传输只在时隙边界处开始。

  • 利用率：37%

前身：CSMA

  1. 监听是否有其他站点传输，如果有就等
  2. 空闲了就传
  3. 等待确认
  4. 没收到确认就重传

在帧传输时间远大于传播时间的网络中很有效

非持续CSMA

  1. 如果空闲，转第二步
  2. 忙就等，转第一步，等待的时间长度是从一个概率分布里面抽取的。
  3. 冲突就规避

1持续CSMA

  1. 空闲就传输，否则转第二步
  2. 忙则继续监听，直到信道空闲，然后立刻传输
  3. 冲突就规避

p持续CSMA

  1. 信道空闲则传输的概率为p，延迟一个时间单位(最大传播时延)再传输的概率为(1-p)
  2. 如果忙就监听，直至空闲，转第一步
  3. 如果已经延迟了一个时间单位，转第一步

CSMA/CD

  1. 如果空闲就传输，否则转第二步
  2. 如果忙就继续监听，直到空闲就立刻传输
  3. 传输过程中检测到冲突，则传输一个干扰信号，保证所有的站点都知道发生冲突并停止传输。
  4. 发送干扰信号后随机等待一个时间，称为退避，然后转第一步。

Notes：最小帧长度
最小帧长度必须要大于争用期传输的bit量，争用期规定为具体是51.2μs51.2\mu s51.2μs，对于10Mb/s10Mb/s10Mb/s的局域网，在争用期内可以传输的bit数量为512个，所以也可以说争用期是512比特时间。

6.4 流量与差错控制*

6.4.1 停等

最简单的流量控制形式，实体在接受到帧之后，返回一个确认，而原点在发送下一个帧之前必须等待，知道接受到这个确认。

  • 缺点：当数据率很高，活着发送方与接收方之间的距离非常远的时候，停等流量控制的线路利用率不高。

6.4.2 滑动窗口

滑动窗口允许一次传送多个帧，这可以大大提升效率。在滑动窗口中，每个帧都通过一个序号来标识。而返回的确认帧表达的是下一个希望接受到的帧号。因此A维护一张允许发送的序号列表，B维护一张它准备接受的序号列表。

每当一个帧发出去的时候，窗口会变小，每当确认一个帧的时候，窗口就会变大。
现举例说明

  1. A发送F0，F1，F2，并暂时缓存这三个帧，窗口向右缩小到3开始。目的站将窗口向右缩小到3开始，并返回RR3，表示我希望接受序号为3的帧。
  2. A接收到RR3，丢弃缓存的三个帧，将窗口向右扩展三个，继续发送F3，F4，F5，F6。。。

6.4.3 停等ARQ

源点站设有一个定时器，接收站通过差错检测技术找到差错的存在并简单丢弃这个帧，所以，源点计时器如果超时就会重传刚才的帧。

停等ARQ基于停等流量控制协议，它在源点传输一个帧之后，必须等待一个确认。以下为停等ARQ的过程

  1. 帧损伤：这个时候是发送方的帧没有到达接收方，于是接收方不会发送确认，所以会超时导致重传。
  2. ACK损伤：这个时候接收方的ACK信息没有到达接收方，于是发送方再次发送上一个帧，接收方在接受后将丢弃重复的帧并再次发送刚才的ACK信息。注意，为了区分帧的重复，帧是交替编码的，即帧0和帧1.

6.4.4 返回N ARQ

基于滑动窗口的差错控制。如果终点检测到差错，需要发送一个否认，且终点丢弃这个帧以及所有后续接受到的帧，直到有差错的帧被正确收到，所以源点收到否认之后必须重传有差错的帧以及这个帧之后的所有已经传输过的帧。

现假设B已成功接受到前面的帧i−1i-1i−1，并且A刚刚传输了iii

  1. 帧损伤：接收到的帧有问题，接收站丢弃该帧，接下来会有两种情况。

    • 合理时间范围内，A发送帧(i+1)，B接收到了帧(i+1)发现次序不对，发送REJ i，此时A必须重传i及以后的所有帧。
    • A没有马上发送其他帧，B没有接收到任何帧，并且B不返回RR和REJ，这个时候会超时，这时候A会发送一个RR信号(包括特殊命令)要求B发送自己想要的下一个帧。

  2. RR损伤：RR发送回A的时候出现了问题

    • B接收到帧i并发送RR(i+1)，但它在传输时丢失，由于确认时累计的，所以A可能会接收到下一个帧的RR，且这个帧在计时器超时前到达。
    • A的计时器超时了，它会传输一个RR命令(包含特殊命令)，但会启动一个p计时器，如果B不响应RR就会超时，超时多次将启动复位。

  3. REJ损伤：REJ损伤同1(b)

6.4.5 选择拒绝ARQ

特点就是只重传那些接收到否认的帧活着超时的帧，但这要求双方要有更复杂的控制逻辑，让算法变得复杂，所以人们可能更倾向于使用N ARQ。对于卫星链路这种传播时延很长的，选择拒绝是一种好的选择。

7.1.3 IP地址

分类的IP地址
IP地址分为A，B，C，D，E五类，其中ABC三类是单播地址，是最常用的

  • A类：0+网络号，共八位，24位主机号。
  • B类：10+网络号，共16位，16位主机号。
  • C类：110+网络号，共24位，8位主机号。
  • 网络号：标志主机连接到的网络，一个网络号在整个互联网范围内必须是唯一的
  • 主机号：标志该主机或路由器，主机号在网络号指明的网络范围内必须是唯一的。
  • 所以IP在整个互联网范围内是唯一的

划分子网
使用三级划分
{<网络号>,<子网号>,<主机号>}
由于32位IP地址本身以及数据报的首部都没有包含任何有关子网划分的信息，所以必须使用子网掩码来进行区分。

构造超网CIDR
无分类编址，IP地址变为
{<网络前缀>,<主机号>}
它还使用斜线记法，比如128.14.35.7/20表示前20位是网络前缀。

7.1.4 ARP

ARP通常划分到网络层，它是用来探测已知IP地址的MAC地址的。步骤如下

  1. 在本局域网上广播一个ARP请求分组
  2. 本局域网上的所有主机上运行的ARP进程都收到此ARP分组
  3. 主机手下ARP请求分组，向A发送响应分组，在这里面写入自己的MAC地址
  4. A收到B的ARP响应分组之后，在其ARP高速缓存里面写入B的IP地址到MAC地址的映射。

高速缓存里面的每一个映射地址都有一个生存时间(10-20min)，因为网络总是在变化的，所以需要删除一些存在时间长的。

7.1.5 IPv4与IPv6的对比

IPv6主要提供了一些增强功能，并且地址范围大得多。它为了适应现代网络的图像和视频流，但真正驱动大家开发IPv6的动力是对更多地址的需求。IPv6包含了128bit的源地址和目的地址字段！
